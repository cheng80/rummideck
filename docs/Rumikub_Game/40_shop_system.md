# 40_shop_system.md

## 1. 목적
이 문서는 Balatro의 **상점 시스템(The Shop)**을 그대로 유지하면서, 프로젝트에서 **Jester 중심 구조로 재정의된 상점 시스템**을 정의한다.

> 원칙: **구조 / 확률 / 가격 / 리롤 규칙은 원본 유지, 제거된 시스템만 비활성화**

## 1.1 현재 구현 기준 메모 (2026-04-06)

- 현재 MVP 구현은 Blind별 고정 보상 대신, **스테이지 클리어 시 `10 + (남은 Plays × 5)` Gold** 를 즉시 지급한다.
- 현재 리롤은 **$5에서 시작해 리롤마다 +$1 증가**한다.
- 새 상점은 `ShopState` 인스턴스를 다시 만들기 때문에 리롤 비용이 **$5로 리셋**된다.
- 현재 런타임 상점은 `Anomaly` 타입을 쓰고 있으며, Balatro 카피 기준의 최종 목표는 `Jester` 카탈로그 기반 상점이다.

---

# 2. 상점 기본 구조

## 2.1 진입 타이밍
- Balatro 원형은 **각 Blind 승리 후 상점 진입**이다.
- Small / Big 스킵은 Tag 보상과 연결된다.
- Boss Blind 이후에도 동일하게 Shop을 거친 뒤 다음 Ante로 넘어간다.

---

## 2.2 기본 구성 (원본 기준)

Balatro 상점은 다음 요소로 구성된다:

- 랜덤 카드 2개 (Joker / Tarot / Planet 중 등장)
- Booster Pack 2개
- Voucher 1개

---

## 2.3 프로젝트 적용

이번 프로젝트에서는 다음을 제거한다:

- Tarot ❌
- Planet ❌
- Spectral ❌
- Voucher ❌
- Booster Pack ❌

👉 결과:

**MVP 상점 = Jester 전용 상점**

> 단, 원형 복제 기준 문서에서는 Voucher / Planet / Booster가 “제거된 시스템”이 아니라 **추후 복구 가능한 비활성화 상태**로 본다.

# 3. Jester 등장 구조

## 3.1 원본 확률 (참고)

| 타입 | 가중치 |
|------|--------|
| Joker | 20 |
| Tarot | 4 |
| Planet | 4 |

👉 약 70% Joker

---

## 3.2 프로젝트 적용

- 모든 슬롯 = Jester
- 확률 제거 → 100% Jester 등장

---

# 4. 슬롯 구조

## 4.1 기본 슬롯 수

- 기본: 2개
- Jester 제안 슬롯 기준

---

## 4.2 확장 가능

- 향후 확장 시 슬롯 증가 가능

---

# 5. 리롤 (Reroll)

## 5.1 기본 규칙

- 시작 비용: $5
- 리롤 시 +$1 증가
- 새 상점 진입 시 초기화

---

## 5.2 동작

- 모든 Jester 슬롯 재생성

---

# 9. 핵심 설계 의미

상점은 단순 보상이 아니라:

👉 **빌드 방향을 결정하는 시스템**

플레이어는 상점에서:
- 어떤 Jester를 선택할지
- 어떤 전략을 갈지

결정한다.

# 10. 설계 원칙

### 원칙 1
상점은 항상 선택을 강제해야 한다

### 원칙 2
항상 최적 선택이 보이면 안 된다

### 원칙 3
리롤은 전략적 선택이어야 한다

### 원칙 4
Jester 조합이 게임을 결정해야 한다
