# 70_playtest.md

## 1. 목적
이 문서는 Balatro 구조를 기반으로 한 본 프로젝트의 **게임성 검증 및 밸런스 테스트 기준**을 정의한다.

👉 목표:
- 게임이 "재미있는지" 검증
- 밸런스 붕괴 여부 확인
- Jester 중심 플레이가 정상 동작하는지 확인

---

# 2. 테스트 원칙

## 원칙 1
한 판의 재미를 검증한다

## 원칙 2
수치보다 체감 우선

## 원칙 3
Seed 기반 재현 필수

---

# 3. 핵심 검증 질문

1. 손패 8개만으로 직관적 플레이 가능한가
2. 족보 선택이 즉시 이해되는가
3. Jester가 플레이를 바꾸는가
4. Stage 난이도가 자연스럽게 증가하는가
5. 다시 플레이하고 싶은가

---

# 4. 테스트 시나리오

## 4.1 기본 플레이
- 10~20 런 진행
- 랜덤 Seed + 고정 Seed 혼합

---

## 4.2 빌드 테스트

### Triple 중심
- 안정적 점수 확인

### Straight 중심
- 고점 가능성 확인

### 혼합
- Jester 의존도 확인

---

## 4.3 난이도 테스트

- Stage 1: 거의 성공
- Stage 3: 빌드 필요
- Stage 5: 선택 중요

---

# 5. 로그 기록

각 런마다 기록:

- Seed
- Stage 도달
- 최종 점수
- 평균 점수
- 사용 족보
- 사용 Jester

---

# 6. 자동 테스트

## 목표
- 100 런 시뮬레이션

## 봇 유형

### 안정형
- Triple 우선

### 공격형
- Straight 우선

### 균형형
- 최대 점수 선택

---

# 7. 실패 케이스

## 족보 문제
- Triple만 쓰는 게 정답
- Straight가 안 쓰임

## Jester 문제
- 특정 Jester만 사용됨

## 점수 문제
- 초반부터 어려움
- 후반 운빨

---

# 8. 성공 기준

다음 중 4개 이상 만족:

- 규칙 이해 쉬움
- 빌드 차이 존재
- Jester 영향 체감
- 난이도 자연스러움
- 재플레이 의사 있음

---

# 9. Seed 테스트

- 동일 Seed → 동일 결과
- 입력 동일 시 재현

---

# 10. 설계 원칙

### 원칙 1
테스트는 반복한다

### 원칙 2
결과는 기록한다

### 원칙 3
감이 아니라 데이터로 판단

---

# 11. 금지사항

- 테스트 없이 수치 변경 ❌
- 감으로 밸런스 조정 ❌

---
